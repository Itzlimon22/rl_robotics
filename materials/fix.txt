Bug 4: framelinvel/frameangvel sensors return BODY frame, not world frame (auv.xml + auv_env.py)
Your XML uses framelinvel and frameangvel with objtype="body". In MuJoCo 3.x, these return velocity in the body frame by default. But in _apply_fluid_forces() and _get_observation() you treat them as world frame and rotate with R.T. This double-rotates the velocity.
Check by adding this sanity test after reset:
pythonv_sensor = env.data.sensordata[7:10]   # what the sensor gives
v_world = env.data.qvel[0:3]           # ground truth world velocity
print("sensor:", v_sensor)
print("qvel:  ", v_world)
If they differ significantly when the AUV is rotated, you have this bug. The fix is either: use env.data.qvel[0:3] directly for world-frame velocity, or remove the R.T rotation in _apply_fluid_forces since the sensor already gives body-frame. Pick one consistent convention and apply it everywhere.
This needs verification before you trust any physics.
